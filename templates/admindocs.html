API documentation
For all requests, headers must be parsed in the format {"api_key": YOUR_API_KEY_HERE} for authentication.

GET http://127.0.0.1:5000/api/v1/licenses/{license}
------------------
Retrieves information relating to the license parsed into the parameters of the request, denoted by {license}. If the request authenticates successfully, a JSON object of the specfic license will be returned.
POST http://127.0.0.1:5000/api/v1/licenses/{license}
REQUIRES body paramaters of "HWID" and "device"
------------------
Updates the HWID and devicename for a given license, specified in the request. Both can be set to None in the case where the user is attempting to log out (unbind) of their device. If the request is valid and authenticated successfully, it will return a JSON object of the updated license and a suitable status.
